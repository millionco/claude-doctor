# claude-optimizer

Analyze Claude Code transcripts for quality signals, sentiment drift, and behavioral anti-patterns. Generates AGENTS.md rules from your session history.

## Install

```bash
npm i -g claude-optimizer
```

## Usage

```bash
# Check health of your latest session
claude-optimizer

# Check a specific project
claude-optimizer -p myproject

# Check a specific session file
claude-optimizer -s ~/.claude/projects/.../SESSION_ID.jsonl

# Analyze all sessions across all projects
claude-optimizer --all

# Generate AGENTS.md rules from your history
claude-optimizer --rules

# Save model to .claude-optimizer/ (for hook integration)
claude-optimizer --save

# JSON output (for piping to hooks)
claude-optimizer --json
claude-optimizer --all --json
```

## What it detects

Reads `~/.claude/projects/**/*.jsonl` transcripts and runs three layers of analysis:

### Lexical

AFINN-165 sentiment scoring with custom agent-specific tokens (corrections, profanity, frustration keywords).

### Structural

| Signal | What it catches |
|---|---|
| `edit-thrashing` | Same file edited 5+ times in one session |
| `error-loop` | 3+ consecutive tool failures |
| `excessive-exploration` | Read-to-edit ratio above 10:1 |
| `restart-cluster` | Multiple sessions started within 30 minutes |
| `high-abandonment-rate` | Most sessions end with fewer than 3 messages |

### Behavioral

| Signal | What it catches |
|---|---|
| `correction-heavy` | 20%+ of user messages are corrections ("no", "that's not", "wait") |
| `keep-going-loop` | User repeatedly says "keep going" / "continue" |
| `repeated-instructions` | User says the same thing twice (Jaccard similarity >60%) |
| `negative-drift` | Messages get shorter and more corrective over time |
| `rapid-corrections` | User responds within 10 seconds of agent output |
| `high-turn-ratio` | User sends 1.5x+ messages per agent response |

## Generating AGENTS.md rules

```bash
claude-optimizer --rules
```

Outputs ready-to-paste rules derived from your actual session history:

```markdown
## Auto-generated rules

Based on analysis of 838 sessions. Generated by claude-optimizer.

- Read the full file before editing. Plan all changes, then make ONE complete edit. If you've edited a file 3+ times, stop and re-read the user's requirements.
- After 2 consecutive tool failures, stop and change your approach entirely. Explain what failed and try a different strategy.
- When the user corrects you, stop and re-read their message. Quote back what they asked for and confirm before proceeding.
- Complete the FULL task before stopping. If the user asked for multiple things, implement all of them before presenting results.
```

## Saving the model

```bash
claude-optimizer --save
```

Writes to `.claude-optimizer/`:

- `model.json` — signal baselines and project profiles
- `guidance.md` — agent-readable rules

The `guidance.md` can be included in your AGENTS.md or injected via Claude Code hooks.
